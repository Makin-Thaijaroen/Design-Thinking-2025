# Design-Thinking-2025

เพื่อช่วยแก้ปัญหากลุ่มคนที่ไม่มีความรู้ความเข้าใจเกี่ยวกับค่าความดันโลหิต(SYSTOLIC mm Hg (top/upper number),DIASTOLIC mm Hg (bottom/lower number))ในการตรวจสอบว่าค่าความดันโลหิตต่างๆที่ได้มาจะอยู่ในกลุ่มไหน

## 🚀 ฟีเจอร์หลัก (Features)
1) ระบบประเมินระดับความดันโลหิตอัตโนมัติ
   - ผู้ใช้กรอกค่า Systolic (ตัวบน) และ Diastolic (ตัวล่าง)
   - ระบบจะประเมินระดับความดันให้อัตโนมัติ
2) โหลดข้อมูลระดับความดันจากไฟล์ CSV
   - ใช้ไฟล์ที่ได้จากการ scraping
   - ดึงชื่อระดับความดันจากคอลัมน์
3) ตรวจสอบค่าที่ไม่สมเหตุสมผล
   - ถ้าใส่ค่าความดันเป็น 0 จะขึ้นว่า Invalid (ค่าความดันไม่สมเหตุสมผล / ไม่ถูกต้อง)
   - ถ้าไฟล์ไม่มี CSV ในการดึงชื่อระดับความดันจะขึ้น Error แล้วหยุดทำงาน
4) จำแนกระดับความดัน 6 ระดับ
   - ระบบสามารถแยกได้ดังนี้ (🚨 Hypertensive Crisis (วิกฤต),🚨 Stage 2 Hypertension,⚠️         Stage 1 Hypertension,📈 Elevated,⚠️ Low Blood           Pressure,✅ Normal)
5) แสดงผลแบบมีสีและคำแนะนำ
   - ระบบจะใช้สีเป็นการบ่งบอกความอันตรายของระยะความดัน
   - พร้อมกับให้คำแนะนำสุขภาพเบื้องต้นตามระดับที่ตรวจพบ

## 💡 สะท้อนคิด
| นายพรรษกร เหล่าสูง | นายเมฆินทร์ ไทยเจริญ |
| :--- | :--- |
| จากการทำโครงงานครั้งนี้ ข้าพเจ้าได้เรียนรู้การเขียนโค้ด Web Scraping และการพัฒนาเว็บแอปพลิเคชันด้วยภาษา Python รวมถึงการใช้งาน GitHub เพื่อจัดการโค้ดและทำงานร่วมกับผู้อื่น นอกจากนี้ยังได้ฝึกการค้นคว้าข้อมูลที่เกี่ยวข้อง การแบ่งงานตามที่ตกลงกัน และการวางแผนทำงานเป็นทีม
ในส่วนที่ควรพัฒนาเพิ่มเติม คือการศึกษาข้อมูลและเทคนิคการเขียนโค้ดให้ลึกขึ้น โดยเฉพาะการเขียนเงื่อนไขให้รอบคอบมากขึ้น เนื่องจากพบว่าการเรียงลำดับข้อมูลในไฟล์ CSV มีผลโดยตรงต่อการทำงานของเว็บแอป หากมีการเปลี่ยนแปลงอาจทำให้ผลลัพธ์คลาดเคลื่อนได้ อีกทั้งควรพัฒนาทักษะการเลือกใช้ข้อมูลที่มีคุณภาพและเชื่อถือได้มากขึ้น
โดยรวมแล้ว โครงงานนี้ช่วยพัฒนาทั้งทักษะด้านเทคนิคและการทำงานร่วมกับผู้อื่น ซึ่งเป็นประสบการณ์ที่มีประโยชน์และสามารถนำไปต่อยอดในอนาคตได้ | สิ่งที่ผมได้สะท้อนคิดจากโปรเจกต์นี้คือ การทำ Web Scraping ไม่ใช่แค่การเขียนโค้ดคัดลอกข้อความ แต่คือการสร้าง 'ระบบอัตโนมัติ' ที่ทำงานแทนคนอย่างรอบคอบครับ ผมได้เรียนรู้ตั้งแต่การเขียนคำสั่งให้บอทรู้จัก 'รอ' หน้าเว็บโหลดเพื่อป้องกันโปรแกรมพัง การมองหน้าเว็บให้เป็นโครงสร้าง HTML เพื่อเขียนลูปเจาะเข้าไปดึงข้อมูลที่ซับซ้อนอย่างตารางได้อย่างแม่นยำ และบทเรียนที่สำคัญที่สุดคือ 'การทำความสะอาดข้อมูล (Data Cleaning)' เพราะข้อมูลดิบบนเว็บมักจะมีขยะหรือสัญลักษณ์แปลกๆ ปะปนมาเสมอ งานนี้ทำให้ผมเข้าใจว่า หัวใจหลักของการสกัดข้อมูล ไม่ใช่แค่การดึงมาให้ได้เยอะๆ แต่คือกระบวนการเปลี่ยนข้อมูลดิบที่กระจัดกระจาย ให้กลายเป็นชุดข้อมูลที่สะอาด มีโครงสร้างชัดเจน และพร้อมนำไปใช้งานต่อได้ทันที |
